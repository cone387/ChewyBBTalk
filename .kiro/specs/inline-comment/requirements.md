# 内联评论功能需求

## 概述
在碎碎念卡片（BBTalkCard）底部直接展示评论区域，用户无需跳转页面即可查看和发表评论。评论功能定位为"自回复/追加内容"，类似微信朋友圈的评论交互。

## 用户故事

### US-1: 查看评论
- 作为用户，我希望在碎碎念卡片底部直接看到评论内容
- 如果有评论，卡片底部显示评论列表（最多显示 3 条，超过 3 条显示"查看全部 N 条评论"）
- 每条评论显示：用户名 + 评论内容 + 时间，单行紧凑排列

### US-2: 发表评论
- 作为用户，我希望点击卡片底部的评论按钮，弹出底部输入框（BottomSheet/Modal），输入内容后发送
- 发送成功后，评论立即出现在卡片底部，评论计数 +1
- 输入框支持多行，最大 500 字

### US-3: 删除评论
- 作为用户，我希望长按自己的评论可以删除
- 删除前弹出确认弹窗
- 删除成功后评论从列表移除，评论计数 -1

### US-4: 展开全部评论
- 作为用户，当评论超过 3 条时，我希望点击"查看全部"可以展开所有评论
- 展开后显示"收起"按钮

## 非功能需求

### NFR-1: 性能
- 评论数据懒加载：首次渲染卡片时不加载评论内容，只显示评论计数
- 用户点击评论按钮或"查看全部"时才请求评论列表
- 评论列表缓存在组件 state 中，避免重复请求

### NFR-2: 交互
- 点击卡片内容区域仍然进入编辑（恢复原有行为）
- 评论按钮放在卡片 footer 区域（时间旁边）
- 底部输入框弹出时自动聚焦键盘

## 不做的事情
- 不做详情页（BBTalkDetailScreen 删除）
- 不做评论嵌套/回复评论
- 不做评论编辑
- 不做评论点赞
