# 快速修复 iOS 图标空白问题

## 问题原因
当前的 PNG 图标文件有透明背景（RGBA），iOS 会将透明部分显示为空白。

## 解决步骤（5分钟搞定）

### 1. 打开在线工具
在浏览器中打开：`frontend/generate-ios-icon.html`

### 2. 上传当前图标
- 点击"选择图标文件"
- 上传 `frontend/public/pwa-icon.png`（这是你的源图标）

### 3. 选择背景色
- 点击颜色选择器
- 输入你想要的背景色（建议使用品牌主色）
- 或者保持默认的蓝色 #4F46E5

### 4. 调整图标大小
- 拖动"图标缩放"滑块
- 建议设置为 80%，这样图标周围会有适当的边距
- 在预览中查看效果

### 5. 下载新图标
点击三个下载按钮，分别下载：
- `apple-touch-icon.png` (180x180) - iOS 主屏幕图标
- `pwa-192x192.png` (192x192) - Android PWA 图标  
- `pwa-512x512.png` (512x512) - 高清图标

### 6. 替换文件
将下载的文件复制到 `frontend/public/` 目录，覆盖原有文件

### 7. 重新构建
```bash
cd frontend
npm run build
```

### 8. 部署并测试
1. 部署应用
2. 在 iOS 设备上删除旧的 PWA 图标
3. 重新添加到主屏幕
4. 检查图标是否正常显示（没有空白边距）

## 提示
- 背景色建议使用品牌主色，保持视觉一致性
- 图标缩放建议 70-85% 之间，太大会显得拥挤，太小会显得空旷
- 生成的图标是纯色背景（无透明），iOS 会自动添加圆角效果
- 如果还有其他尺寸的图标需要生成，可以重复使用这个工具

## 如果你想批量生成所有尺寸
可以安装 sharp 库后使用脚本：
```bash
cd frontend
npm install --save-dev sharp
node fix-ios-icon.cjs
```

但手动使用在线工具更直观，可以实时预览效果。
